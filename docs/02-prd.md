# PRD — Pause (가명)

작성일: 2026-07-15
버전: v0.1 (프로토타입 기획 단계)
근거 문서: `01-research-summary.md`

## 1. 한 줄 정의

부부·연인이 다투다가 감정이 격해지는 그 순간, 함께 켜는 실시간 대화 도우미. 매일 쓰는 관계개선 퀴즈가 아니라, 위기의 순간에만 쓰는 "브레이크" 도구.

## 2. 문제 정의

기존 커플 커뮤니케이션 앱(Paired, Gottman Card Decks 등)은 전부 평상시에 쓰는 예방적 도구다. 그러나 커뮤니티 리서치 결과, 관계에 실제로 타격을 주는 순간은 평온한 저녁이 아니라 이미 감정이 격해진 찰나였다 ("술 취한 밤이 결혼을 망칠 뻔했다", "새벽 5시 대치" 등). 이 순간을 위한 도구가 시장에 사실상 없다 (Harmony 등 극소수 초기 시도만 존재).

또한 기존 앱들의 공통 실패 원인은 명확하다:
- 한쪽만 참여해서 무용지물이 되는 "채택률" 문제
- "숙제처럼 느껴진다"는 피로감
- 부부 이중 과금에 대한 반감

## 3. 타겟 사용자

- 1차: 이미 어느 정도 관계 개선 의지가 있는 커플(둘 다 "우리 사이 좋아지고 싶다"는 데 동의한 상태) — 완전히 냉담하거나 이혼을 확정한 관계는 대상 아님
- 2차 확장: 시댁/처가 갈등, 육아 스트레스로 인한 갈등을 겪는 기혼 커플 (한국 시장 로컬라이징 시 우선순위)

### 명시적 비대상
- 가정폭력·정신건강 위기 상황 — 이 앱은 치료·상담을 대체하지 않으며, 해당 신호 감지 시 전문기관 연결 안내로 전환해야 함 (기획 초기부터 명시)

## 4. 핵심 가치 제안 (Value Proposition)

| 기존 앱 | Pause |
|---|---|
| 평온할 때 쓰는 예방적 퀴즈 | 격해지는 순간에 쓰는 반응형 개입 |
| 매일 능동적 입력 필요 → 피로감 | 세션 켜기 전까지는 아무 입력도 요구하지 않음 |
| 부부 이중 과금 | 1구독 = 커플 둘 다 이용 |
| 텍스트 정확도에 의존 | 음향 신호(음량·속도·겹침) 우선 판단 → 겹치는 말에도 강건 |

## 4-1. 설계 원칙: 극단적 단순함 (Radical Simplicity)

2026-07-15 논의에서 확정: **Pause는 "그 순간의 개입" 단 하나만 하는 단일 목적 도구다.** Harmony를 포함한 경쟁 앱들이 세션 예약, 퀴즈, 저널, 점수화 등으로 범위를 넓혀온 것과 정반대 방향으로 간다.

기준 문장: *"화가 난 상태에서 이 화면을 보고도 3초 안에 뭘 눌러야 하는지 헷갈린다면, 그 기능은 뺀다."*

이 원칙은 향후 기능 추가 논의 시 1차 필터로 사용한다.

### 안 만들 것 (Explicit Non-Goals)

Harmony 앱 실측(2026-07-15, "New Session" 화면 기준)에서 확인된 요소들과 대비:

| Harmony에 있는 것 | Pause의 결정 | 이유 |
|---|---|---|
| Solo 모드 / With Partner 토글 | 만들지 않음 — 항상 둘이 쓰는 것만 존재 | 혼자 쓰는 모드는 "그 순간 개입"이라는 목적과 무관, 곧 범위 확장의 씨앗이 됨 |
| Chat / Voice 모드 선택 | 만들지 않음 — 음성만 | 선택지 자체가 화난 상태에서의 마찰. 음성이 자연스러운 싸움 방식에 가장 가까움 |
| 세션 시간 선택 (5/10/15/20/30/45/60분) | 만들지 않음 — 시작만 있고 끝은 사용자가 "종료" 누를 때 | 싸움은 정해진 러닝타임이 없음. 시간 선택 자체가 실사용 맥락과 안 맞음 |
| "파트너 초대 대기" 온보딩 플로우 | 최소화 — 페어링은 최초 1회만, 매 세션마다 반복하지 않음 | 매번 초대·대기하는 구조는 순간적 개입의 목적을 무너뜨림 |
| Daily Dashboard / 무드 체크인 | 만들지 않음 | 예방적 일일 습관 도구는 이미 Paired 등이 있고, 이 자리에서 검증하려는 문제가 아님 |
| Private Journal | 만들지 않음 | 별도 문제(자기 성찰)이며 v1 범위 밖 |
| Shared Goals / Connection Score | 만들지 않음 | 게임화·점수화는 "숙제처럼 느껴진다"는 기존 앱의 실패 패턴을 반복할 위험 |
| Love Style 퀴즈 / 온보딩 퀴즈 | 만들지 않음 | 첫 사용 전 마찰을 늘리는 요소. 개입 로직은 퀴즈 없이도 동작해야 함 |

### 이상적인 v1 플로우 (수정)

기존 6절 플로우에서 아래를 더 단순화 (2026-07-15 두 기기 결정 반영):
- 시간 선택 없음, Solo/Voice/Chat 선택 없음 — 그 외엔 그대로 최소화 유지
- 파트너 페어링은 앱을 처음 함께 쓸 때 1회만 수행 (예: 링크 공유 후 자동 연결)
- 세션 시작은 "각자 한 번씩" — A가 "시작", B가 "참여" — 매번 초대·대기 화면을 새로 띄우지 않고, 이미 페어링된 상대를 바로 세션에 연결
- 세션 종료도 버튼 하나 ("끝내기") — 요약 화면은 보여주되 추가 입력 요구 없음

## 5. MVP 범위 (v1 — 웹 프로토타입)

### 포함 (In scope)
1. **세션 시작 (두 기기)**: 2026-07-15 결정 — v1부터 각자 폰으로 참여하는 **두 기기 구조**로 간다 (기술 스펙 참고, 겹치는 말 문제를 아키텍처로 우회). 한 명이 "시작" → 상대는 자기 폰에서 "참여" 한 번 탭 → 마이크 권한 요청, 녹음 중임을 명확히 표시. 완전한 원버튼은 아니지만 각자 한 번씩, 총 두 번의 탭으로 끝남
2. **1단계 감지 (음향 레벨)**: 각자 기기에서 자신의 음량·발화 속도를 로컬로 계산 + 두 기기의 발화 타이밍을 비교해 "동시에 말하고 있음" 여부 판단 → 임계치 초과 시 "지금 대화 온도가 올라가고 있어요, 잠깐 멈춰볼까요?" 같은 중립적 넛지 표시 (둘 다에게 동일하게 표시 — 한쪽만 지적하지 않음)
3. **2단계 감지 (의미 레벨)**: 각 기기가 주로 자기 목소리를 선명하게 잡으므로, 굳이 "겹치지 않는 구간"만 기다리지 않고도 각자 발화를 STT로 텍스트 변환 가능 → 비난조 표현 감지 시 "이렇게 말해볼까요?" 리프레이밍 제안 (Gottman의 "비난→욕구 표현" 원칙 기반)
4. **세션 종료 후 요약**: 감정 온도 그래프, 오갔던 리프레이밍 제안 목록을 둘 다 볼 수 있는 요약 화면
5. **동의/프라이버시 화면**: 세션 시작 전 양쪽 모두 "녹음에 동의함"을 명시적으로 확인받는 플로우 (법적 요건이자 신뢰 요건)

### 제외 (Out of scope, v1)
- 상시 백그라운드 감청 (OS 정책상 불가, 리서치 문서 6절 참고)
- 생체 데이터(수면·심박) 연동
- 네이티브 앱 전환
- **푸시 알림을 통한 자동 참여** — 상대 기기가 백그라운드에 있으면 OS가 마이크를 막기 때문에, 알아서 몰래 켜지는 방식은 애초에 불가능. v1은 상대가 직접 "참여"를 눌러야 하는 것을 전제로 함 (말로 알리거나 화면 알림 정도만 지원, 자동 시작 아님)
- 다국어 지원 (v1은 한국어 우선)

## 6. 핵심 사용자 플로우 (2026-07-15 두 기기 결정 반영)

1. 다투기 시작 → 한 명(A)이 "Pause 켤까?" 제안 → 자기 폰에서 앱 열고 "시작" 탭
2. 상대(B)도 자기 폰에서 앱 열고 "참여" 탭 (말로 알리는 정도, 자동 알림은 v1 범위 아님)
3. 양쪽 다 동의 화면에서 "동의" 클릭 → 마이크 권한 허용 → 세션 시작 (양쪽 화면에 녹음 중 표시 상시 노출)
4. 대화 중 각자 기기의 음향 신호(음량/속도) + 두 기기 간 발화 타이밍 비교로 격화 감지 시 화면에 중립적 넛지 표시
5. 각자 기기가 자신의 목소리를 주로 포착하므로 겹침을 기다릴 필요 없이 비난조 표현 감지 시 리프레이밍 제안 카드 표시 (원할 때만 참고, 강제 아님)
6. 대화 종료 → 아무 쪽이나 "세션 종료" 클릭 → 양쪽 다 요약 화면 제공 → 원하면 저장/삭제 선택

## 7. 성공 지표 (프로토타입 단계)

정량 지표보다 초기 정성 검증이 우선이나, 참고 지표:
- 세션 완료율 (시작 대비 끝까지 진행한 비율)
- 재사용 의향 (세션 후 "다음에도 쓸래?" 자체 응답)
- 양쪽 다 세션에 참여했는가 (채택률 문제 검증의 핵심 지표)
- 정성 인터뷰: "이 순간에 실제로 앱을 켤 것 같은가?"가 가장 중요한 검증 질문

## 8. 리스크 및 대응

| 리스크 | 대응 |
|---|---|
| 화난 상태에서 앱을 켤 마음 자체가 안 생김 | 초기 사용자 인터뷰로 "켜는 행위" 자체의 마찰을 최우선 검증 |
| 오탐(false positive)으로 신뢰 상실 | 1단계(음향)는 보수적 임계치로 시작, 2단계(의미)는 참고용 카드로만 제공하고 강제 개입 지양 |
| 한쪽이 "감시당하는" 느낌을 받음 | 판정자가 아닌 중재자 톤 유지, 항상 양쪽에게 동일한 메시지 노출 (한쪽만 지적 금지) |
| 법적/프라이버시 리스크 | 세션 시작 전 양쪽 명시적 동의 필수, 원본 오디오 비영구 저장(세션 종료 시 삭제 옵션 기본값) |
| 가정폭력 등 위험 신호 오분류 | 위기 신호 감지 시 일반 넛지 대신 전문기관 연결 안내로 분기 (상세 기준은 후속 문서에서 정의) |

## 9. 차별화 근거 요약

- Harmony와 컨셉은 유사하나(Harmony도 각자 기기로 참여하는 실시간 세션으로 추정), Pause는 세션 예약·퀴즈·저널·점수화 없이 "그 순간의 개입" 단 하나에만 집중하는 단일 목적 도구 (4-1절 극단적 단순함 원칙)
- 두 기기 구조를 채택해 "동시 발화" 문제를 기술로 어렵게 풀기보다 아키텍처로 우회 — 기술 리스크를 낮추고 프로토타입 속도를 우선함 (2026-07-15 결정)
- LoveFix류의 "사후 복구"가 아니라 "그 순간" 개입에 집중 — 리서치에서 확인된 미해결 니치
- 커플 1구독 정책으로 기존 앱들의 이중과금 반감을 원천 차단
- 한국어 화법(존댓말 붕괴, 시댁/처가 관련 표현) 특화는 해외 경쟁 앱들이 다루지 않는 영역
